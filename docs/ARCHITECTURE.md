# Архитектура «Звонилка»

## Обзор

Приложение построено по модульной архитектуре с чётким разделением ответственности.
Каждый JS-файл — отдельный модуль (ES Module) с одной зоной ответственности.

## Диаграмма модулей

```
┌─────────────────────────────────────────┐
│               index.html                │
│         (загружает app.js)              │
└────────────────┬────────────────────────┘
                 │
          ┌──────▼──────┐
          │   app.js    │  ← Точка входа
          │ (оркестратор)│
          └──┬───┬───┬──┘
             │   │   │
     ┌───────┘   │   └───────┐
     ▼           ▼           ▼
┌─────────┐ ┌──────────┐ ┌─────────┐
│  ui.js  │ │signaling │ │webrtc.js│
│         │ │   .js    │ │         │
└────┬────┘ └────┬─────┘ └────┬────┘
     │           │            │
     └───────────┴────────────┘
                 │
          ┌──────▼──────┐
          │  config.js  │  ← Настройки
          │  utils.js   │  ← Утилиты
          └─────────────┘
```

## Поток данных при звонке

1. **Пользователь** нажимает «Позвонить» → `ui.js`
2. `ui.js` → `signaling.js`: создать комнату в Supabase
3. `signaling.js` → `webrtc.js`: запросить микрофон, создать offer
4. `webrtc.js` → `signaling.js`: отправить offer в Supabase
5. **Второй пользователь** получает offer через Supabase Realtime
6. `webrtc.js` создаёт answer → `signaling.js` отправляет answer
7. Обмен ICE-кандидатами через `signaling.js`
8. **WebRTC PeerConnection** установлено → голос идёт напрямую (P2P)

## Масштабирование

### Добавить новый модуль
1. Создать файл в `src/js/`
2. Экспортировать `initМодуль()` функцию
3. Импортировать и вызвать в `app.js`

### Добавить новые стили
1. Создать файл в `src/css/`
2. Подключить через `<link>` в `index.html`

### Добавить звуки / иконки
1. Положить в `assets/sounds/` или `assets/icons/`
2. Обновить список `STATIC_ASSETS` в `sw.js`
