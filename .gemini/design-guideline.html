07 — Анимации

Движение и переходы

Четыре анимации на весь продукт. Каждая несёт смысловую нагрузку — не декорация.

📞
Pulse Glow — Главная кнопка в Idle
Пульсирующее свечение вокруг кнопки. Сигнализирует «живой, готов к звонку». Работает пока нет активного состояния.
2s ease-in-out ∞
Ripple Rings — Состояние Calling
Концентрические кольца расходятся от кнопки. Буквально — как сигнал вызова распространяется к маме.
1.5s ease-out ∞ staggered
…
Fade Blink — Статус «Соединяю»
Плавное мигание текста статуса. Не мигает резко — именно fade. Показывает ожидание без тревоги.
1.5s ease-in-out ∞
02:47
Screen Transition — Смена состояний
Плавный fade (opacity 0→1) при смене экранов. 250ms — достаточно быстро чтобы не раздражать, достаточно медленно чтобы не пугать маму.
250ms ease
prefers-reduced-motion
Все анимации выключаются если пользователь включил системную опцию «Уменьшить движение». Обязательно для доступности.
@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; } }
08 — Правила дизайна

Делать / Не делать

Конкретные решения, принятые один раз навсегда.

✓ Делаем
  • Одна большая кнопка по центру — мама нажимает, не думая
  • Тёмный фон везде — экран не слепит ночью
  • Цвет кнопки меняется со статусом (фиолет → янтарь → зелёный)
  • Ошибки объясняют что делать, а не что сломалось
  • Эмодзи вместо SVG — ноль веса, максимальная понятность
  • safe-area-inset — кнопки не прячутся под вырезом iPhone
  • 100dvh — высота корректная на Safari с адресной строкой
  • Хаптик при входящем — рука чувствует, уши могут не слышать
✗ Не делаем
  • Светлая тема — не нужна, усложняет поддержку
  • Больше одной кнопки на экране — кроме Incoming (там 2)
  • Анимации длиннее 400ms — раздражает при частом использовании
  • Технические ошибки напрямую: «ICE failed», «WebRTC error»
  • Загрузка шрифтов через @import — каждая мс на счету в PWA
  • Gradient text везде — только для заголовков дизайн-доки
  • Notification permissions без контекста — сначала пусть позвонит
  • border-radius: 0 — это не продукт для разработчиков
Тест «Правило мамы»
Перед любым дизайн-решением — один вопрос: «Мама разберётся за 3 секунды?»
Если нет — убираем, упрощаем или объясняем. Аудитория — не молодой разработчик, а человек, которому нужно просто позвонить сыну.
